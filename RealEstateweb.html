JWT Authentication Flow

Secure, stateless authentication for all operations

1

User Registration

User creates account with email and password. System hashes password using bcrypt with salt rounds and stores securely in MongoDB database.

2

Login & Token Generation

User submits credentials. Server validates against database, generates JWT token with user ID and role payload, signs with secret key.

3

Token Storage

Frontend receives token and stores in localStorage or httpOnly cookie. Token persists across sessions for seamless authentication.

4

Request Authorization

Every API request includes token in Authorization header. Frontend automatically attaches token to all protected route requests.

5

Middleware Verification

Backend middleware intercepts requests, verifies token signature, validates expiration, and checks user role permissions.

6

Access Control

If token valid and role authorized, request proceeds to route handler. Otherwise, returns 401 Unauthorized error response.
